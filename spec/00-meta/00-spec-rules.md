# Specification Rules

## 1. 문서의 권위 (Authority)
* 이 폴더(`/spec`) 내의 모든 문서는 코드보다 상위의 권한을 가진다.
* 코드와 스펙이 충돌할 경우, **스펙을 기준으로 코드를 수정**하거나, **스펙 변경 절차(ADR)**를 밟아야 한다.
* AI(LLM)는 코드를 작성하기 전 반드시 관련 도메인 및 기능 스펙을 먼저 숙지해야 한다.

## 2. 언어 및 용어 (Language & Terminology)
* 비즈니스 로직과 변수명은 `/spec/00-meta/02-glossary.md`에 정의된 영문 용어를 사용한다.
* UI 텍스트는 한국어를 기본으로 하되, 톤앤매너는 `/spec/60-ux/copy-guidelines.md`를 따른다.

## 3. 구현 원칙 (Implementation Rules)
* **모호함 해결:** 스펙에 명시되지 않은 에러 케이스나 엣지 케이스는 '사용자의 심리적 안전감'을 최우선으로 하여 방어적으로 구현한다.
* **데이터 무결성:** `InsightMe`는 사용자의 인생 기록이므로 데이터 유실은 허용되지 않는다. 로컬 DB(Room/Realm)와 리모트 동기화 시 충돌 해결 전략은 '최신 수정'보다 '데이터 병합'을 우선한다.
