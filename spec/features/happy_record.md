# 행복 기록 기능 명세서

## 🎯 기능 개요
사용자가 일상의 행복한 순간을 간편하게 기록하고, 이를 통해 개인의 행복 패턴을 파악할 수 있는 기능

## 📋 주요 기능 (MVP)

### ✍️ 행복한 순간 작성 화면
**목적**: 행복한 순간을 쉽고 빠르게 기록할 수 있는 사용자 친화적 인터페이스 제공

**주요 입력 항목**:
- **감정 강도** (필수): 1-5점 스케일 슬라이더
- **상황 태그** (선택): 미리 정의된 태그 선택 (가족, 친구, 업무, 취미 등)
- **구체적 내용** (선택): 자유 텍스트 입력
- **시간** (자동): 현재 시간 자동 입력, 수동 변경 가능
- **위치** (선택): GPS 기반 위치 정보

**UX 요구사항**:
- 값 입력 시 다음 필드로 자동 포커스 이동
- 저장 CTA 버튼은 하단에 항상 노출
- 필수값 미입력시 명확한 안내 메시지

### 📋 기록 목록 보기 화면
**목적**: 작성한 행복 기록들을 효과적으로 탐색하고 관리

**표시 형태**:
- 타임라인 형태의 카드뷰
- 각 카드에 날짜, 감정 강도, 주요 태그, 내용 일부 표시

**필터 및 정렬**:
- **정렬**: 날짜순(최신/오래된), 감정 강도순
- **필터**: 특정 태그별, 감정 강도 범위별, 기간별
- **검색**: 내용 키워드 검색

**인터랙션**:
- 카드 탭시 상세 보기/편집 화면 이동
- 스와이프 제스처로 빠른 삭제

### 📊 감정 통계 요약 화면
**목적**: 기록된 데이터를 바탕으로 개인의 행복 패턴 인사이트 제공

**통계 항목**:
- **기본 통계**:
  - 주간/월간 기록 수
  - 평균 감정 강도
  - 가장 행복했던 날과 그 이유
  
- **패턴 분석**:
  - 자주 등장하는 단어/태그 워드클라우드
  - 요일별 행복도 분포
  - 시간대별 행복 기록 패턴

- **트렌드 분석**:
  - 월별 감정 강도 변화 그래프
  - 특정 태그의 빈도 변화

**시각화**:
- 직관적인 차트와 그래프 활용
- 색상 코딩으로 감정 강도 구분
- 인터랙티브한 차트 (터치로 상세 정보 확인)

## 🏗️ 기술적 구현사항

### 데이터 모델
```dart
class EmotionRecord {
  String id;
  DateTime timestamp;
  int emotionIntensity; // 1-5
  List<String> tags;
  String? content;
  RecordLocation? location;
  List<RecordPhoto>? photos;
}
```

### 주요 화면 플로우
1. **홈 화면** → **빠른 감정 점수 선택** → **상세 작성 화면**
2. **목록 화면** → **필터/정렬** → **개별 기록 상세보기**
3. **통계 화면** → **기간 선택** → **상세 분석 보기**

### 성능 요구사항
- 기록 작성 완료까지 10초 이내
- 목록 화면 로딩 시간 2초 이내
- 월 단위 데이터 시각화 렌더링 3초 이내

### 오프라인 지원
- 네트워크 연결이 없을 때도 기록 작성 가능
- 연결 복구시 자동 동기화

## 🎨 UI/UX 가이드라인

### 홈 화면 디자인
**구조**: 스크롤 가능한 세로 레이아웃
```
Column {
  [최근 작성한 기록 패널]
  [명언 패널] 
  [여백]
  [새 기록 작성 패널]
}
```

**최근 작성 기록 패널**:
- 화면 상단 가로 채움, 내용 가운데 정렬
- 최대 5개 기록, 내용은 최대 3줄까지
- 좌우 스와이프로 네비게이션
- 기록이 없을 경우 "빨리 기록하라"는 안내 문구

**새 기록 작성 패널**:
- **Idle 상태**: 웃는 얼굴 아이콘의 "새 기록 버튼"
- **Selecting 상태**: 1-5점 점수 슬라이더
- touchDown시 버튼 → 슬라이더로 애니메이션 전환
- 점수 선택 후 손을 떼면 해당 점수로 작성 페이지 이동

### 접근성
- 색약 사용자를 위한 패턴/텍스트 병용
- 스크린 리더 지원을 위한 적절한 레이블링
- 최소 터치 영역 44px 확보
