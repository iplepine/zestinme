# 인사이트 리포트 스펙

## 목적

인사이트 리포트는 사용자가 지난 상태를 길게 분석하게 만드는 화면이 아니라 다음 주에 조정할 행동을 정리해 주는 요약이다.

## 입력 데이터

- 최근 7일 `ConditionRecord`
- 최근 7일 `SleepRecord`
- 개입 결과 이벤트
- 선택 입력 태그와 메모

## 기본 구성

- 한 줄 요약: 이번 주 컨디션 흐름
- 좋았던 조건: 에너지/회복이 높았던 날의 공통점
- 주의 신호: 반복된 낮은 회복, 높은 스트레스, 낮은 집중
- 다음 행동: 다음 주에 시도할 행동 1개
- 신뢰도: 데이터 부족 시 낮은 확신 표현

## 문구 원칙

- 진단처럼 말하지 않는다.
- 원인 단정 대신 관찰과 제안을 분리한다.
- 데이터가 부족하면 부족하다고 말한다.
- 사용자가 바로 실행할 수 있는 행동 1개로 끝낸다.

## MVP 범위

MVP에서는 홈의 `이번 주 한 줄` 수준으로 제한한다. 별도 장문 리포트 화면은 후순위다.
