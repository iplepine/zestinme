# 범위 정리

## MVP에 포함

- 온보딩
- 첫 컨디션 체크인
- 홈의 `오늘 상태`, `지금 할 일`, `이번 주 한 줄`
- 30초 컨디션 체크인
- 회복 로그 저장/수정
- 월간 타임라인
- 최소 1개의 규칙 기반 개입 결과 기록
- 로컬 우선 저장

## MVP에서 숨김 또는 보류

- `/dev` 중심 진입
- 로그인 중심 플로우
- Firebase Auth 의존 플로우
- 서버 백업
- 소셜 공유
- 장문 AI 상담
- 외부 수면 기기 연동
- 웨어러블/폰 사용/캘린더 데이터 의존
- 다중 인사이트 카드 대시보드

## 레거시로만 해석

- Mind-Gardener 세계관
- 식물 종 배정 결과
- GardenRepository 중심 확장
- Self-Mirror
- Ventilation
- `garden`, `seeding`, `caring` 네이밍

## 정리 원칙

- 코드가 남아 있어도 MVP 메인 플로우에서 적극 노출하지 않는다.
- 새 기능은 핵심 루프를 닫는 데 직접 기여할 때만 포함한다.
- 수면 저장은 단일 write path로 수렴해야 한다.
