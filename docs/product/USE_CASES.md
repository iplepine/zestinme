# 핵심 유스케이스

## UC1. 첫 실행 후 첫 체크인

- 사용자는 앱을 처음 열고 닉네임을 입력한다.
- 첫 컨디션 체크인을 저장한다.
- 저장이 완료된 뒤 홈으로 진입한다.
- 완료 기준: 온보딩 미완료 사용자가 홈을 먼저 보지 않는다.

## UC2. 오늘 상태 확인

- 사용자는 홈에서 오늘 상태를 확인한다.
- 앱은 최신 컨디션, 오늘 회복 로그, 최근 7일 기록을 바탕으로 상태를 계산한다.
- 데이터가 부족하면 확신이 낮은 표현과 첫 행동 CTA를 보여준다.
- 완료 기준: 사용자가 오늘 무엇을 먼저 해야 하는지 1개만 본다.

## UC3. 30초 체크인

- 사용자는 에너지와 회복만으로 빠르게 체크인을 저장할 수 있다.
- 집중, 스트레스, 태그, 상황, 몸 반응, 메모는 선택 입력이다.
- 저장 후 홈과 타임라인에 즉시 반영된다.
- 완료 기준: 저장 성공/실패 상태가 분명하다.

## UC4. 회복 로그 남기기

- 사용자는 취침 시간, 기상 시간, 수면 질 또는 회복감을 기록한다.
- cross-day 시간은 정상화한다.
- 기존 기록은 수정할 수 있다.
- 완료 기준: 홈과 타임라인이 같은 저장 데이터를 다시 읽는다.

## UC5. 지난 흐름 확인

- 사용자는 타임라인에서 날짜별 컨디션과 회복 로그를 함께 확인한다.
- 최근 7일 기준 한 줄 요약을 본다.
- 기록 삭제/수정 후 집계가 다시 계산된다.
- 완료 기준: 기록 존재 여부와 날짜별 상세 흐름이 일관된다.

## UC6. 개입 결과 반영

- 사용자는 홈의 단일 CTA 또는 최소 개입에 반응한다.
- 앱은 `completed`, `dismissed`, `ignored`, `snoozed` 중 하나로 결과를 저장한다.
- 같은 목적의 개입은 사용자의 반응 이후 반복 노출이 줄어든다.
- 완료 기준: 제안이 노출만 되고 사라지지 않고 다음 판단에 반영된다.
