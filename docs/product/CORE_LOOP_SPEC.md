# 핵심 루프 스펙

## 루프

```text
첫 진입 -> 첫 체크인 -> 홈 상태 판독 -> 행동 1개 제안 -> 행동/기록 저장 -> 다음 판독에 반영
```

## 필수 단계

1. 온보딩 진입
2. 닉네임 입력
3. 첫 컨디션 체크인 저장
4. 홈에서 오늘 상태와 지금 할 일 확인
5. 체크인 또는 회복 로그 추가 저장
6. 타임라인에서 기록 확인
7. 개입 결과 이벤트 저장

## 홈 판단 규칙

- 오늘 체크인이 없으면 `오늘 체크인`을 우선 제안한다.
- 회복 로그가 없거나 회복 상태가 낮으면 `회복 로그 남기기`를 우선 제안한다.
- 데이터가 부족하면 추정 표현을 낮추고 첫 행동 CTA를 보여준다.
- 동시에 여러 주요 CTA를 노출하지 않는다.

## 성공 기준

- 계정, 서버, 외부 권한 없이 루프가 완주된다.
- 체크인과 회복 로그는 같은 로컬 저장 기준으로 다시 조회된다.
- 사용자가 다음 행동을 고르는 데 드는 판단 비용이 낮다.
