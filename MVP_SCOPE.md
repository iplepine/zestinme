# MVP 현실화 정리

작성일: 2026-04-16  
목표: 감성 콘셉트보다 실제로 동작하는 핵심 기능을 먼저 완성한다.

## 1. MVP 한 줄 정의

ZestInMe MVP는 사용자가 감정과 수면을 기록하고, 기록을 다시 확인하며, 일정 시간이 지난 감정에 대해 짧은 회고를 남길 수 있는 로컬 우선 자기돌봄 앱이다.

식물과 정원은 MVP에서 핵심 기능이 아니라 상태를 보여주는 간단한 시각적 피드백으로 제한한다.

## 2. MVP 핵심 루프

MVP에서 반드시 완성해야 하는 루프는 아래 4개다.

```text
1. 오늘 감정 기록
2. 오늘 또는 어젯밤 수면 기록
3. 기록 목록/캘린더에서 다시 확인
4. 일정 시간이 지난 감정에 짧은 회고 답변
```

이 4개가 안정적으로 동작하면 MVP로 볼 수 있다.

## 3. MVP 포함 기능

### 3.1 감정 기록

반드시 포함:

- 감정 좌표 또는 감정 태그 선택
- 메모 입력
- 저장
- 저장 성공/실패 피드백
- 오늘 기록 또는 최근 기록 확인

보류:

- 복잡한 감정 분석
- AI 기반 긴 코칭
- 감정별 정원 날씨 세분화
- 고급 심리 모델 UI

### 3.2 감정 회고

반드시 포함:

- 기록 후 4시간 이상 지난 감정 찾기
- 짧은 질문 1개 제시
- 답변 저장
- 회고 완료 여부 표시

보류:

- 3단계 이상 깊이의 코칭 플로우
- 개인화 질문 추천 고도화
- 보상 연출
- Firestore 기반 코칭 기록 동기화

### 3.3 수면 기록

반드시 포함:

- 잠든 시간 또는 침대에 누운 시간 입력
- 기상 시간 입력
- 수면 질 점수 입력
- 아침 상쾌함 점수 입력
- 메모 또는 태그 입력
- 최근 30일 조회
- 간단한 차트 또는 리스트

보류:

- 수면 효율 고급 계산
- 멀티터치 차트 확대/축소
- 복잡한 툴팁 편집
- 수면 패턴 분석 문장 생성

### 3.4 히스토리

반드시 포함:

- 월별 감정 기록 조회
- 날짜별 기록 목록
- 기록 상세 확인
- 수면 기록 접근 또는 별도 최근 수면 목록

보류:

- Monthly Garden 시각화
- 식물 아카이브
- 정체성 카드
- 통합 인사이트 리포트

### 3.5 온보딩

반드시 포함:

- 최초 실행 여부 저장
- 닉네임 또는 최소 프로필 값 저장
- 홈 진입 가능 상태 만들기

보류:

- 환경값 기반 식물 종 배정 고도화
- 복잡한 모듈 선택
- 감성적인 온보딩 장면 확대

## 4. 식물/정원 MVP 범위

식물은 디자인 비용이 크기 때문에 MVP에서 다음처럼 축소한다.

반드시 포함:

- 대표 식물 이미지 1개 사용
- 홈 화면에서 현재 상태 표시
- 감정 기록 버튼 또는 홈 CTA와 연결
- 성장 단계는 이미지 교체 없이 크기, 투명도, glow, 흔들림 정도로만 표현

보류:

- 식물 여러 종 이미지
- 성장 단계별 이미지
- 화분 이미지 다양화
- 식물 도감
- 희귀도 시스템
- 꽃말 기반 상세 연출
- 식물별 custom layout 대량 조정

현재 코드의 `PlantDatabase`와 `PlantService`는 유지해도 되지만, MVP 화면에서는 대표 식물 1종만 보여준다. 이미지가 준비되기 전까지 식물 종 다양성은 사용자 기능으로 노출하지 않는다.

## 5. 저장소 기준

MVP에서는 저장소를 단순하게 가져간다.

권장 기준:

- 감정 기록: Isar
- 온보딩: Isar
- 수면 기록: Isar로 통합
- Firebase: 질문 은행 Remote Config 정도만 선택적으로 사용

현재 수면 기록은 Hive Repository 흐름과 Isar `LocalDbService` 흐름이 함께 있다. MVP 안정화를 위해 수면 기록 저장/조회/수정/삭제를 한 저장소로 통일해야 한다.

## 6. MVP에서 제거하거나 숨길 것

아래 항목은 코드가 일부 있어도 MVP 화면에서 적극적으로 노출하지 않는다.

- `/dev` 초기 진입
- 로그인 화면
- 복잡한 GardenRepository 설계
- 식물 DB 이전
- 식물 다종 배정 결과 표시
- 발견/인사이트 탭
- Self-Mirror
- Ventilation
- 소셜 공유
- 푸시 알림
- 서버 백업
- Firebase Auth
- 장기 개인화 AI 코칭

## 7. 우선순위 작업 목록

### 1순위: 앱 진입 플로우 정리

목표:

- 일반 실행 시 `/dev`가 아니라 온보딩 또는 홈으로 진입
- 개발자 화면은 직접 경로로만 접근

완료 기준:

- 첫 실행: 온보딩 표시
- 온보딩 완료 후: 홈 표시
- 재실행: 홈 표시

### 2순위: 수면 저장소 일원화

목표:

- 수면 기록의 저장/조회 경로를 하나로 통일

완료 기준:

- 수면 추가, 수정, 삭제, 최근 30일 조회가 같은 저장소를 사용
- 관련 테스트 통과
- Hive와 Isar 중복 흐름 제거 또는 명확한 마이그레이션 TODO 작성

### 3순위: 감정 기록 저장 UX 안정화

목표:

- 감정 기록이 저장됐는지 사용자가 분명히 알 수 있게 한다.

완료 기준:

- 저장 중 상태 표시
- 저장 실패 시 메시지 표시
- 저장 성공 후 홈 또는 히스토리에서 바로 확인 가능

### 4순위: 회고 기능 최소 완성

목표:

- 감정 기록 후 시간이 지난 기록에 질문 1개를 붙이고 답변 저장

완료 기준:

- 회고 대상 기록 조회
- 질문 표시
- 답변 저장
- 회고 완료 기록은 다시 대상에 나오지 않음

### 5순위: 식물 MVP 축소 반영

목표:

- 식물 이미지 디자인 부담 없이 홈의 상태 피드백 역할만 수행

완료 기준:

- 대표 식물 1개로 고정
- 없는 asset을 참조하지 않음
- 성장 단계는 scale/effect로만 표현
- 식물 종 다양성 UI 제거 또는 숨김

## 8. MVP 완료 기준

아래 조건을 만족하면 MVP 1차 완성으로 본다.

- 새 사용자가 앱을 켜고 온보딩을 마칠 수 있다.
- 홈에서 감정 기록을 추가할 수 있다.
- 홈 또는 히스토리에서 기록을 확인할 수 있다.
- 수면 기록을 추가하고 최근 기록을 볼 수 있다.
- 감정 기록 일부를 회고 완료 처리할 수 있다.
- 앱 재실행 후 기록이 유지된다.
- `flutter analyze`와 핵심 테스트가 통과한다.

## 9. 명확히 하지 않을 것

MVP에서는 앱을 “완성된 감성 정원”처럼 보이게 만드는 데 시간을 쓰지 않는다.

하지 않을 것:

- 식물 asset 대량 제작
- 장면별 일러스트 추가
- 감성 문구 대량 작성
- 복잡한 애니메이션
- 개인화 인사이트 문장 생성
- 도감/희귀도/수집 시스템

먼저 사용자가 기록하고, 다시 보고, 짧게 회고하는 기능을 안정화한다.
